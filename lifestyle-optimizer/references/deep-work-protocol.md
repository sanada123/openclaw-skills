# Deep Work Protocol — Extended Reference

Based on Cal Newport's *Deep Work* (2016) + Andrew Huberman's focus protocols.

## The Science

- **Ultradian rhythm**: 90-minute cycles of focus → 20-minute recovery. Work WITH this, not against it.
- **Attention residue** (Leroy, 2009): Switching tasks leaves cognitive residue for ~23 minutes. Every context switch costs you 23 minutes of recovery.
- **Flow state entry**: Takes 15-25 minutes of uninterrupted focus to enter flow. Any interruption resets the clock.

## Protocol Levels

### Level 1: Beginner (Week 1-2)
- One 60-minute deep work block per day
- Phone on silent in the same room
- Goal: complete one meaningful task per session

### Level 2: Intermediate (Week 3-6)
- Two 90-minute blocks per day (morning + afternoon)
- Phone in another room
- Calendar blocked, Slack status set to DND
- Goal: 10-12 hours of deep work per week

### Level 3: Advanced (Week 7+)
- Three 90-minute blocks per day
- Separate deep work environment (different desk, library, café)
- All notifications disabled system-wide during blocks
- Goal: 15-20 hours of deep work per week (world-class)

## Environment Design

| Factor | Why | How |
|--------|-----|-----|
| Same location | Trains associative memory | Designated "deep work chair" or spot |
| Same time | Reduces startup friction | Brain enters ready state automatically |
| Visual cue | Signals intent | Noise-canceling headphones = "I'm working" |
| Clean desk | Reduces cognitive load | Only current task materials visible |

## Measuring Success

Track daily:
```
Date: ___
Deep work hours: ___
Main output: ___
Interruptions: ___
Energy after: 1-10
```

Weekly: sum hours, identify patterns, adjust.

## Common Failure Modes

1. **Planning too many blocks** — Start with 1/day. Build up.
2. **No pre-commitment** — If you don't decide what to work on BEFORE the block, you'll drift.
3. **Skipping shutdown** — Without explicit closure, work bleeds into evening.
4. **Checking "just one thing"** — One Slack check = 23 minutes of attention residue.
